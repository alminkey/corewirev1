# CoreWire Media-Tech Reset Design

**Date:** 2026-04-12  
**Status:** Approved for planning  
**Scope:** Replace the current redesign direction with a more radical `modern media-tech platform` identity across the public site and owner admin.

## Why This Reset Exists

The previous redesign improved layout and cleanliness, but it did not create a fundamentally new product feeling.

It still reads as:

- a restyled news site
- a lighter version of the old product
- a visual iteration on the same concept

That is not enough.

The target now is not:

- newspaper on the web
- premium magazine layout
- retro newsroom shell

The target is:

- a modern digital product for intelligence and news

## Product Decision

CoreWire should look and feel like a `modern media-tech platform`.

This means:

- product-grade hierarchy
- stronger contrast and surfaces
- clearer modular sections
- bolder navigation and layout zoning
- less classic news typography dominance
- less “article page as paper page”
- more “information platform with premium editorial content”

This is a full concept change, not another polish pass.

## Core Visual Direction

### Identity

CoreWire should feel like:

- a serious intelligence and news platform
- modern, fast, deliberate
- premium through product quality, not nostalgia

It should not feel like:

- newspaper heritage brand
- classic magazine
- terminal theme
- SaaS dashboard with articles attached

### Typography

The typographic center shifts away from old-paper serif styling.

Rules:

- strong modern sans for major UI and many headlines
- serif becomes selective and secondary, not dominant
- headings should feel sharp, digital, and product-grade
- metadata and navigation should look contemporary and clean

### Color and surfaces

The system should move toward:

- cooler, cleaner base neutrals
- stronger ink/dark surfaces where needed
- brighter contrast between modules
- a sharper accent system

The palette should feel more like a high-end product interface than a warm newspaper page.

Accent behavior:

- signal orange / copper may remain
- but used as product accent, not editorial paper tint

### Motion

Motion remains subtle, but more product-like:

- sharper hover response
- panel elevation
- cleaner transitions between states
- no decorative nostalgia

## Public Frontend

### Homepage intent

The homepage should feel like a live intelligence product, not a front page from a publication template.

It should communicate:

- urgency
- structure
- authority
- modernity

### Homepage structure

The homepage should be rebuilt around:

1. `top product bar`
   - strong brand
   - edition/state chips
   - clean section navigation

2. `platform hero`
   - high-contrast lead module
   - more like a product hero than a newspaper lead
   - strong CTA and clear status

3. `live desk / signal rail`
   - fast-scanning update modules
   - clearer hierarchy than current right rail

4. `modular story surfaces`
   - cards that feel like platform modules
   - less “story card on paper”
   - stronger visual grouping

5. `media block`
   - optional embedded video / signal brief / explainers

### Article pages

Article pages should still be highly readable, but not look like classic editorial pages.

Desired feeling:

- premium reading environment
- cleaner, more product-like rhythm
- less decorative editorials
- stronger content framing around flagship analysis

Key outcomes:

- stronger headline block
- better section zoning
- cleaner full-article body container
- source/facts sections that feel integrated into a platform

## Owner Admin

### Admin intent

The admin should feel like a `control plane`.

Not:

- a blog admin
- a long landing page
- a lighter article layout

It should feel closer to:

- a product dashboard
- an operational workspace
- a newsroom system console with modern UX discipline

### Admin structure

The admin remains a shell, but visually it must move further into product territory:

- darker or stronger sidebar zone is acceptable
- top utility chips / pills
- denser, clearer stat modules
- cleaner editing canvas
- stronger separation between:
  - queue
  - drafts
  - publishing
  - programming

### Admin design rules

- cards should feel like product modules, not article panels
- typography should be cleaner and less editorial than the public site
- the editor should feel like a real application workspace
- navigation should feel deliberate and fast

## Shared System Rules

Even though admin and frontend differ in tone, they must clearly belong to one product.

Shared:

- brand mark
- accent system
- UI radius/shadow language
- buttons and form style
- surface hierarchy
- motion rules

Different:

- frontend leans public/editorial
- admin leans operational/product

## What Changes Compared to the Previous Redesign

The previous redesign emphasized:

- lighter theme
- serif editorial hierarchy
- warmer surfaces
- clearer but still familiar news layout

The new reset emphasizes:

- modern digital product feel
- stronger UI modularity
- less traditional news-site composition
- less classic editorial styling
- more media-tech identity

## Non-Goals

This reset does not yet include:

- backend changes
- new content model
- Paperclip UI
- new homepage data model
- motion-heavy interactions

This is a visual and structural product reset on the existing frontend/admin system.

## Release Outcome

This reset is complete when:

- the homepage no longer reads as a lightly restyled newspaper page
- the admin no longer reads as a dressed-up blog control page
- both surfaces feel like parts of one modern information platform
- the product finally looks materially different from what existed before
